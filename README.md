# ⏰ Auto Ticket — Плагин для FunPay Cardinal

## 📋 Описание

**Auto Ticket** — плагин для [FunPay Cardinal](https://github.com/sidor0912/FunPayCardinal), который автоматически находит неподтверждённые заказы и отправляет тикеты в техподдержку FunPay с просьбой их подтвердить.

Плагин позволяет автоматизировать рутинное общение с поддержкой, самостоятельно группируя старые заказы и создавая обращения по заданному расписанию.

## 🌟 Возможности

| Функция | Описание |
|---------|----------|
| 🎫 Авто-тикеты | Автоматическая генерация и отправка обращений в поддержку FunPay |
| 🔍 Превью | Просмотр списка заказов, подходящих под критерии, перед отправкой |
| 🛡 Лимиты | Ограничение количества создаваемых тикетов в сутки для защиты аккаунта |
| ⏳ Гибкий возраст | Настройка минимального времени жизни заказа перед созданием тикета |
| 🔄 Дедупликация | Система запоминает отправленные заказы и не пишет по ним повторно |
| ⚙️ Управление через Telegram | Полная настройка всех параметров через inline-меню бота |
| 🔁 Автоповтор | До 3 попыток обращения к API при сетевых сбоях или ошибках сервера |
| 💾 Персистентность | Настройки и база отправленных ID сохраняются между перезапусками |

## 📦 Установка

1. Скачайте файл `auto_ticket.py` из этого репозитория.
2. Откройте чат с ботом FunPay Cardinal в Telegram.
3. Введите команду: `/upload_plugin`.
4. Отправьте скачанный файл плагина боту.
5. Перезапустите бота.

> **Примечание:** Плагин не требует установки дополнительных зависимостей — используются только стандартные модули FunPay Cardinal и библиотека `beautifulsoup4`.

## ⚙️ Настройка

### Быстрый старт

1. Откройте Telegram-бота FunPay Cardinal.
2. Введите команду: `/auto_ticket`.
3. Настройте возраст заказа кнопкой ⏳ **Старше: N ч.** (например, 24).
4. Включите автоматический режим кнопкой ✅/❌ **Авто**.
5. Готово — плагин начнёт проверять и отправлять заказы по расписанию.

### Пример сообщения в поддержку

Здравствуйте! Прошу подтвердить заказы, ожидающие подтверждения:
#O12345, #O67890.
С уважением, ВашНик!

## 🎛 Управление через Telegram

### Команда

/auto_ticket

Открывает меню настроек плагина.

### Кнопки меню

| Кнопка | Описание |
|--------|----------|
| 🔗 Отправить заказы | Запустить цикл проверки и отправки тикетов немедленно |
| 🔍 Превью заказов | Посмотреть список заказов, по которым будут созданы тикеты |
| ✅/❌ Авто | Включить или выключить автоматическую отправку по расписанию |
| 🔔/🔕 Увед. | Включить или выключить уведомления о результатах в Telegram |
| 📋 В тикете: N | Установить макс. кол-во заказов в одном обращении (1–50) |
| ⏳ Старше: N ч. | Установить мин. время с момента оплаты заказа (1–720 ч.) |
| ⏱ N ч. | Установить интервал между автоматическими проверками (1–168 ч.) |
| 🛡 Лимит: N/день | Установить суточный лимит создаваемых тикетов (1–50) |

## 📊 Параметры по умолчанию

| Параметр | Значение | Описание |
|----------|----------|----------|
| Возраст заказа | 24 часа | Заказы младше этого времени игнорируются |
| Заказов в тикете | 10 шт. | Оптимальное количество для одного обращения |
| Интервал авто | 12 часов | Частота фонового сканирования аккаунта |
| Суточный лимит | 5 тикетов | Безопасное количество обращений в сутки |

## 🔧 Технические детали

- **Хранение данных:** Настройки и база отправленных ID сохраняются в:

storage/plugins/auto_ticket/settings.json

- **Защита от спама:** Плагин автоматически сбрасывает счетчик дневного лимита в полночь.
- **Потокобезопасность:** Запись конфига защищена `threading.Lock` для предотвращения повреждения данных.
- **Фоновая обработка:** Автопроверка работает в отдельном потоке и не замедляет работу Cardinal.
- **Безопасная авторизация:** Плагин использует существующую сессию `golden_key`. При разлогине работа приостанавливается.
